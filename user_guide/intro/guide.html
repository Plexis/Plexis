How to Use This Guide

In this page, I will be teaching you how to use the documentation provided here. Learning how an MVC framework works and how its used is not the easiest thing in the world to learn. Below are some examples along with explainations of terms and such that i use through out the guide.

Variable colors

$var - This orange color variable displayed here is used to show that it is a Required variable in that part of the function or class.
$var - This green color variable displayed here is used to show that it is NOT a Required variable in that part of the function or class. In otherwords, it has a default value that will fill in if that variable isnt used by you.

Example use:

$this->load->database('$config_key', '$instance');

In this example above, The $config_key is required. Without that variable, you will mostly see an error pop up when you load the page. On the contrast, $instance is NOT required and the method will work just fine without you setting that variable. It has a default value that will be put there on its absence.

The term: Instance or Instancing

Through out the guide you will often here me mention Intance or Instancing. What this means is, you can use a function to load the super object INTO your current script.

For Example:

$FB = get_instance();
$FB->load->database('DB', TRUE);

Now, what has happened is, you have loaded the Super Object into a variable $FB. You can now use that variable to access all the Libraries, Database connections, and variables that are loaded into the Super Object by going $FB->$class->$method(); instead of loading each class, DB connection, etc etc, manually, and going $this->$class->$method();
